Grâce à l’inspection du code HTML sur la page `index.php?page=media&src=nsa`,
on observe un objet chargé via :
<object data="http://10.11.200.231/images/nsa_prism.jpg"></object>

Modifier le contenu de data par :
data="data:MIME_TYPE;base64,BASE_64_DATA"
nous permet d'intégrer directement une ressource encodée en base64 dans le code html
et en modifier la valeur de `nsa` dans l’URL par la ressource qu'on souhaite.

Donc en tentant une injection, on encode en base64 le script suivant :
<script>alert(1)</script>

On construit ensuite une URL avec le resultat :
src=data:text/html;base64,PHNjcmlwdD5hbGVydCgxKTwvc2NyaXB0Pg==

Après rafraîchissement, le navigateur interprète le contenu encodé et
exécute directement le JavaScript, validant ainsi l’injection côté client.

Recommendations :
- Valider strictement les paramètres côté serveur et n’autoriser que des valeurs prédéfinies (whitelist)
- Échapper ou encoder toute donnée utilisateur avant de l’insérer dans le HTML
- Interdire ou filtrer les schémas dangereux (data:, javascript:, vbscript:)
- Limiter ou éviter l’usage des balises <object> et <embed> si elles ne sont pas nécessaires
- Mettre en place une Content-Security-Policy (CSP) solide pour bloquer
	l’exécution de scripts inline et l’utilisation non autorisée de data:
